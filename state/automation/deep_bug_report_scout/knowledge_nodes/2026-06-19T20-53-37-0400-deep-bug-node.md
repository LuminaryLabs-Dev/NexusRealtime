## Root Lesson
- id: deep-bug-root-2026-06-19-ar-launch-spatial-readmodel
- statement: AR launch capability contracts and SpatialRoom descriptor getters need explicit preconditions, fallback policy, and read-model isolation before they can support reliable browser/AR proof flows.
- why it matters: Current smoke tests pass, but live probes show AR launch can start against a plain engine and crash without `createARKit()`, launcher support state can be mutated through `getState()`, explicit preferred-mode lists silently degrade to fallback, and SpatialRoom read getters expose mutable building/anchor objects.
