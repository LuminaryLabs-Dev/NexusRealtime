# Knowledge Nodes: deep_bug_report_scout 2026-06-19T06:53:41-04:00

## Root Lesson
- id: deep-bug-root-2026-06-19-receipt-identity-idempotency
- statement: Timing, interaction, collectible, and request kits need receipt identity, event idempotency, and generated-id allocation fixtures before they can support promoted action/reward/logistics services.
- why it matters: Current APIs can overwrite action receipt ids, repeat completion and claim events for already-complete state, and create duplicate request ids.
